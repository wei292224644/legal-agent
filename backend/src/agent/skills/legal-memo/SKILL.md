# Legal Memo: Case Analysis

## Purpose

案件分析备忘录是律师/学生思考所在之处。本 skill 提供 IRAC 脚手架并标记研究缺口 — 律师/学生填充分析。

**分析属于律师/学生。** 本 skill  structuring；它不 conclusion。

## Load context

- `practice profile` → 业务领域、管辖、监督风格。
- Intake 摘要和案件笔记用于事实。

## Pedagogy check

读取该业务领域的 supervisor guide（如存在）。检查 `pedagogy_posture` 设置：

- **`guide`（默认）：** 产生 IRAC 结构和研究缺口列表。要求律师/学生自己从研究中起草每个 rule statement，
  而非直接给框架。对他们写的给出反馈。仅在律师/学生尝试一次后提供填充框架 rule。

- **`assist`：** 产生备忘录脚手架并填充可填充的部分。标记需 review 的项目。
  （注意：此 memo skill 始终将 `[STUDENT ANALYSIS]` 和 `[STUDENT CONCLUSION]` 块留空 — 
  `assist` 意味着 skill 产生 IRAC 脚手架和框架 rule statement；它不产生 application 或 conclusion。）

- **`teach`：** 不产生框架或脚手架内容。要求律师/学生自己框定 issue、从研究中陈述 rule、做 application。
  给出反馈。卡住时问引导问题。仅在两次尝试后才展示 model rule statement 或 model application paragraph，
  且仅针对卡住的部分。追踪对错以便 supervisor 查看进度。

如不存在 guide，使用 `guide`。如 guide 存在但未设置 posture，使用 `guide`。

无论 posture 如何，输出始终包含：
"**Pedagogy mode: [assist/guide/teach]** — 由 supervisor guide 设置。这意味着我 [skill 做了什么 vs 律师/学生做了什么]。"

## Workflow

### Step 1: Frame the issues

从 intake 摘要和案件笔记中：本案呈现哪些法律问题？

用问题形式陈述每个。不是"违法解除" — 
"公司以不胜任工作为由解除劳动合同，但未提前三十日书面通知，亦未支付代通知金。
该解除是否构成违法解除，劳动者能否主张 2N 赔偿金？"

如有多项 issue，每个获得自己的 IRAC 块。

### Step 2: Scaffold the IRAC

For each issue:

**Issue:** 来自 Step 1 的问题形式陈述。

**Rule:** 这是研究缺口，不是结论。陈述需要找到的内容：

> `[RESEARCH NEEDED: [地区] 违法解除的认定标准 — 用人单位以不胜任工作解除的法定要件、
> 提前通知义务、代通知金适用条件、赔偿金计算基数上限。
> 从：《劳动合同法》第39-41条、第87条开始，然后查找 [地区] 司法实践中的具体口径。]`

如 skill 对 general rule framework 有较高信心（如"多数地区认可..."），
将其作为 framework starting point 陈述 — **但显式标记为未验证**：

> *Framework（未验证 — 需确认 [地区]）：* 多数地区要求用人单位以不胜任工作解除时须满足：
> 1）有客观的考核标准；2）经培训或调岗；3）提前三十日书面通知或支付代通知金。
> 不符合上述要件可能被认定为违法解除。
> `[VERIFY: [地区] 的具体要件和赔偿金计算基数上限]`

**Application:** 这是律师/学生的分析所在之处。Scaffold 结构，不填充：

> `[STUDENT ANALYSIS: 将规则适用于事实。需要处理的关键事实：
> - 公司主张的不胜任工作具体是什么？有无考核标准？考核是否客观？
> - 公司是否提供了培训或调岗机会？何时、何种形式、有无记录？
> - 解除通知是提前三十日书面送达，还是当天通知当天解除？
> - [地区]-specific: 赔偿金计算基数是否含奖金/津贴？是否有 12 年上限或三倍社平工资上限？
> - 劳动者月工资具体数额？工作年限？]`

列出重要事实。让律师/学生做 applying。

**Conclusion:** 显式留空：

> `[STUDENT CONCLUSION: 基于你的研究和上述分析，可能的结果是什么？
> 这个诉求有多强？弱点在哪里？]`

### Step 3: Identify strengths, weaknesses, open questions

IRAC 块之后的独立部分：

**Strengths（从事实中明显 — 律师/学生应检验这些）：**
- [看似有帮助的事实及原因]

**Weaknesses（从事实中明显 — 律师/学生应评估严重性）：**
- [看似有害的事实及原因]
- `[UNCERTAIN: [X] 是否真的是弱点 — 取决于 [地区] 对 [Y] 的规则]`

**Open questions（备忘录在没有更多信息前无法回答的）：**
- 事实性：[客户处我们不知道什么]
- 法律性：[需要研究的]
- 策略性：[律师/学生/supervisor 的判断 call]

## What this skill does NOT do

- **Write the analysis.** It scaffolds the IRAC and flags the gaps. The attorney/student reasons through the application.
- **Provide verified rules.** Every rule statement is explicitly unverified until the attorney/student researches it.
- **Reach conclusions.** The C in IRAC is blank on purpose.
- **Replace the conversation with the supervisor.** The Open Questions / Strategic section is the agenda for that conversation, not a substitute.
