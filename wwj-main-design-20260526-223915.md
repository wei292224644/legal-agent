## Problem Statement

律师在客户会谈中需要即时调取法规、合同条款和风险提示。目前的做法是：会谈时手动记笔记、会后翻法条、再手动套合同模板——从会谈结束到交付成果可能需要数小时。如果AI能在会谈进行中就实时提供辅助（法规补全、合同条款建议、风险预警），律师当场就能给出更专业的回应，客户体验和成交率都会显著提升。

## What Makes This Cool

实时转录 + AI侧边栏：律师与客户谈话的同时，屏幕上实时显示带角色标签的转写文本，侧边栏AI实时弹出相关法规条文、合同范本条款、风险提示。声纹自动区分律师/客户——不需要任何手动操作。

Demo场景：劳动争议咨询，AI侧边栏实时弹出《劳动合同法》相关条款、赔偿标准、类似判例摘要，最后一键输出合同草案。

## Constraints

- 黑客松时间限制，核心功能必须能在有限时间内跑通
- 中文法律场景，法规引用必须准确（中国法律体系）
- 评委/投资人面向，需要视觉效果和流程体验到位
- 后端Python + 前端Vite+React

## 数据流时序

```
律师开口说话
  │
  ├─ 0ms     MediaRecorder 采集，每800ms发一块到后端
  ├─ 5ms     pyannote VAD 检测到语音活动
  ├─ 100ms   说话人切换→取前1.5秒→pyannote embedding→声纹比对→确认角色
  │          同一说话人继续说→不复核对，沿用已有标签
  ├─ 500ms   千问STT 开始流式返回文字
  ├─ 600ms   前端逐字显示，角色标签已贴上
  ├─ 800ms   句子结束→发送给 Agno Agent
  ├─ 1300ms  deepseek-v4-pro 返回分析 JSON
  ├─ 1400ms  侧边栏卡片滑入
  │
  └─ 端到端延迟：文字 ~600ms / AI分析 ~1.4s（设计目标 <2s）
```

## 技术决策（已确认）

### VAD + 声纹：pyannote-audio（统一依赖）

选择 pyannote 同时处理 VAD 和声纹，不引入额外依赖：

| 环节 | 模型 | 延迟 | 
|------|------|------|
| VAD（检测是否有人说话） | pyannote/voice-activity-detection | ~5ms |
| 声纹提取 | pyannote/embedding | ~50ms |
| 声纹比对 | 余弦相似度 (numpy) | ~1ms |

- 声纹比对仅在**说话人切换时**执行，不每块音频都跑
- 律师声纹预注册（15秒朗读→提取256维向量→存储为 .npy 文件）
- Demo中声纹可预置，现场展示自动角色区分效果

### STT：千问STT API

- 流式语音转文字，中文+法律术语识别
- 音频链路：MediaRecorder(webm/opus) → ffmpeg → 16kHz PCM WAV → 千问STT

### Agent 框架：Agno

- session_state 管理对话上下文（最近10句 + 法律问题摘要）
- @tool 装饰器实现三个核心 Skill：法规查询、合同模板、风险评估
- SqliteDb 持久化 session + memory

### Agent LLM：deepseek-v4-pro

- 流式输出，中文法律理解能力顶级
- 输出格式：JSON（statutes / contract_clauses / risks）

## WebSocket 通信协议

| 方向 | 消息类型 | 负载格式 | 说明 |
|------|---------|---------|------|
| Client → Server | `audio_chunk` | Binary (webm/opus, ~800ms) | 音频切片，带序号 |
| Server → Client | `transcript` | `{"seq":N, "text":"...", "speaker":"律师/客户", "is_final":true}` | 转写结果 |
| Server → Client | `analysis` | `{"seq":N, "category":"statute/contract/risk", ...}` | AI分析 |
| Server → Client | `status` | `{"status":"listening/transcribing/analyzing/error", ...}` | 系统状态 |
| Client → Server | `ping` | `{"type":"ping"}` | 心跳 15s |
| Server → Client | `pong` | `{"type":"pong"}` | 心跳响应 |

## LLM System Prompt

```
你是一位经验丰富的中国执业律师兼法律顾问。你正在实时旁听一场律师与客户的咨询会谈。

任务：
1. 实时识别对话中涉及的法律问题
2. 即时提供相关法规引用（精确到条、款、项）
3. 提供合同条款建议和范本片段
4. 标注潜在法律风险

输出 JSON 格式：
{
  "statutes": [{"law": "法律名称", "article": "第X条", "content": "条文内容", "relevance": "为何相关"}],
  "contract_clauses": [{"title": "条款名", "text": "建议条款文本", "rationale": "建议理由"}],
  "risks": [{"level": "高/中/低", "description": "风险描述", "mitigation": "建议应对"}]
}

规则：
- 只引用中国大陆现行有效的法律法规
- 法规引用必须精确，不确定时标注"[需要核实]"
- 如无相关建议，对应字段返回空数组
```

上下文管理：每轮发送最近10句对话 + 已识别法律问题摘要（滑动窗口 + 摘要），确保不溢出 deepseek-v4-pro 上下文窗口。

- [ ] VAD + 声纹 + STT 管道：说话人切换时自动区分角色
- [ ] 实时转写在屏幕上显示，文字延迟 <1s，角色标签正确
- [ ] AI侧边栏弹出法规、合同、风险三类卡片，延迟 <2s
- [ ] Demo场景完整：5分钟劳动争议咨询，AI全程实时辅助
